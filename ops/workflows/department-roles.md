# ESW Department Roles & Responsibilities

## Layer 0: System Architecture (Meta)

### Agent Architect (HR & Dev)

**Purpose**: Maintains and optimizes the ESW operational system.

| Responsibility | Description |
|---|---|
| Workflow Design | Creates and refines SOPs for each department |
| Quality Control | Audits outputs for consistency and professionalism |
| Specialist Onboarding | Defines role briefs for new hires or temporary specialists |
| Tool Selection | Recommends and implements software/tools for the team |

**Does NOT**: Perform client-facing work.

---

## Layer 1: Operations (The Engine)

### Project Controller (COO/PM)

**Purpose**: Single point of coordination for every project.

| Responsibility | Description |
|---|---|
| Project Intake | Captures client brief, classifies project type |
| Master File | Maintains the single source of truth per project |
| Task Assignment | Breaks projects into phased tasks, assigns to specialists |
| Timeline Management | Tracks milestones, flags delays |
| Client Communication | Primary client contact for status updates |
| Gate Reviews | Approves phase transitions |

**Key Phrase**: _"Awaiting your approval to proceed to Phase [X]."_

---

## Layer 2: Technical Core (The Science)

### Eco-Scientist (Lead Ecologist)

**Focus**: The Problem.

| Responsibility | Description |
|---|---|
| Desktop Studies | Existing data review, species databases, protected area registers |
| Field Surveys | Habitat assessment, species inventory, soil/water sampling |
| EIA Authoring | Environmental Impact Assessment reports per applicable national/international law |
| TNFD Reporting | Nature-related risk and opportunity disclosure |
| Biodiversity Metrics | BNG calculations, habitat condition scoring |
| Peer Review | Reviews Regen-Architect designs for ecological soundness |

**Key References**: IUCN Red List, national/regional protected area databases, applicable EIA legislation, TNFD framework, CSRD, ISSB.

---

### Regen-Architect (Design Lead)

**Focus**: The Solution.

| Responsibility | Description |
|---|---|
| Mitigation Hierarchy | Apply Avoid → Minimize → Restore → Offset framework |
| NbS Design | Nature-based Solutions: swales, buffer strips, pollinator corridors, agroforestry |
| Agrivoltaics | Dual-use solar + agriculture design |
| Hydrology | Water management, constructed wetlands, erosion control |
| Landscape Plans | Spatial design integrating ecology with project infrastructure |
| Restoration Protocols | Species selection, planting schedules, soil amendment plans |

**Key Input**: Eco-Scientist's baseline and risk assessment.
**Key Output**: Design Concept Notes, Restoration Strategies, Spatial Plans.

---

### GIS Analyst (Spatial Data)

**Focus**: The Location.

| Responsibility | Description |
|---|---|
| Site Mapping | Boundary delineation, topography, land cover classification |
| Constraints Analysis | Overlay of protected areas, flood zones, heritage sites, infrastructure |
| Satellite Interpretation | NDVI, land use change detection, seasonal monitoring |
| Spatial Deliverables | Maps, GIS layers, spatial datasets for all departments |
| Data Integration | Combine field data with remote sensing for comprehensive site picture |

**Key Tools**: QGIS, Google Earth Engine, Copernicus data, Landsat, national cadastres and land registries.

---

## Layer 3: Commercial & Legal (The Deal)

### Green Financier (CFO)

**Focus**: The Value.

| Responsibility | Description |
|---|---|
| Ecosystem Valuation | Quantify provisioning, regulating, cultural, supporting service values |
| Carbon Credits | Methodology selection (Verra VCS, Gold Standard), baseline, additionality, forecast |
| Biodiversity Credits | BNG unit calculation, credit pricing, market analysis |
| Green Bonds | Eligibility assessment, use-of-proceeds framework, reporting |
| ROI Modelling | CAPEX vs. yields + credit revenue, IRR, payback period |
| Bankability | Investor memos, risk-adjusted returns, blended finance structures |

**Key Frameworks**: EU Taxonomy, ICMA Green Bond Principles, Verra VCS, Gold Standard, Plan Vivo.

---

### Legal Compliance (General Counsel)

**Focus**: The Risk.

| Responsibility | Description |
|---|---|
| Land Law | Tenure verification, easements, right of access — multi-jurisdictional |
| Permitting | Renewable energy permits, environmental authorization — applicable national frameworks |
| Contracts | Scope of Work, service agreements, credit purchase agreements |
| Liability | Risk exposure assessment, insurance requirements |
| Regulatory Compliance | CSRD, EU Taxonomy, TNFD, ISSB, and national disclosure obligations |
| Due Diligence | Anti-corruption, sanctions screening, beneficial ownership |

**Key References**: Applicable national EIA legislation, CSRD Directive, ISSB Standards, Paris Agreement frameworks, national renewable energy regulations.

---

## Layer 4: Growth (The Hunter)

### Growth Hacker (Sales Lead)

**Focus**: Inbound/Outbound.

| Responsibility | Description |
|---|---|
| Lead Generation | Identify new energy projects, degraded land opportunities, developers |
| Market Intelligence | Track policy changes, subsidy programs, competitor activity |
| Target Qualification | Score leads by budget, timeline, ESW fit |
| Outreach | Cold emails, LinkedIn messages, conference networking |
| Pipeline Management | CRM updates, conversion tracking, weekly lead lists |

**Key Targets**: Solar/wind developers, agribusiness, infrastructure companies, sovereign governments, multilateral agencies, DFIs, impact investors.

---

### Brand Voice (Marketing)

**Focus**: Reputation.

| Responsibility | Description |
|---|---|
| Case Studies | Turn completed projects into client-approved success stories |
| LinkedIn Content | 2-3 posts/week: regulation explainers, project highlights, thought leadership |
| White Papers | Deep-dive documents on NbS, carbon markets, biodiversity credits |
| Website Content | Service pages, blog posts, team bios |
| Investor Materials | Pitch decks, capability statements, ESG credentials |

**Tone**: Visionary, prestigious, scientific. Never salesy. Always backed by data or regulation.
